# summary

關於我

about.md

目標成為專注於伺服器端邏輯與資料層設計的 Java 後端工程師。目前透過一系列由淺入深的練習專案，累積 Spring Boot、Spring Data JPA、關聯式資料庫與快取機制的實作經驗，並持續強化程式碼可讀性、資料一致性與系統除錯的能力。

具備扎實的 Java 語言基礎，熟悉物件導向設計、集合框架與例外處理機制，有助於快速理解程式邏輯與系統架構設計。另外也考取過 CCNA 網路基礎認證，對 TCP/IP、路由與交換器運作原理有一定了解，有助於理解服務之間的通訊架構與部署環境。習慣以文件化方式記錄開發與除錯過程，方便日後回顧、追蹤問題根因，也利於團隊協作與知識交接。

面對環境設定、相依套件衝突等實務問題時，會系統性地拆解排查——這類經驗多次出現在近期專案中，也是我認為後端工程師必備的基本功之一。

# services

專案作品

projects/
書籍管理 REST API 與 Redis 快取優化系統
開發中
Spring Boot Spring Data JPA Hibernate MySQL Redis Docker Compose

以書籍資料為核心的 RESTful API 服務，整合 Redis 快取層以降低資料庫查詢負載，並確保快取與主資料庫之間的一致性。

  • 使用 Spring Boot 與 Spring Data JPA 建立完整 CRUD API，涵蓋新增、查詢、修改、刪除等生命週期。
  • 導入 Cache-Aside 快取策略：查詢優先讀取 Redis，未命中時回源 MySQL 並寫回快取，降低資料庫查詢壓力。
  • 於資料異動（更新／刪除）時同步清除對應快取，避免讀取到過期資料。
  • 提供資料表整表清空（TRUNCATE）與主鍵重置機制，並同步清理關聯快取，確保資料表可重複測試。
  • 以 Docker Compose 整合 Spring Boot、MySQL、Redis 三項服務，建立可一鍵啟動的本地開發環境。
  • 排除多項環境相依問題，包括 JDK 版本衝突、Maven 相依套件錯誤與 Spring annotation 設定錯誤，強化對 Spring Boot 4.x 環境設定與除錯的掌握。
Cache-Aside 讀取流程：命中快取直接回應，未命中時回源 MySQL 並回寫 Redis
